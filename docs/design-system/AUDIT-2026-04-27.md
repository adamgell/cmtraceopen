## Drift between tokens.css and TS themes

| Theme | Token | CSS Value | TS Value |
|-------|-------|-----------|----------|
| classic-cmtrace | `--cmt-fg-1` | `#242424` | `#000000` |
| hotdog-stand | `--cmt-bg-1` | `#FFFF00` | `#FF0000` |
| hotdog-stand | `--cmt-fg-1` | `#000000` | `#FFFF00` |
| nord | `--cmt-fg-1` | `#ECEFF4` | `#D8DEE9` |
| all themes | `--cmt-font-mono` | includes `Cascadia Mono` | omits `Cascadia Mono`, wraps in `var()` |

**Structural risk**: `theme-dark.ts` has zero explicit color overrides — relies entirely on `createDarkTheme(tealBrand)`. The CSS documents specific hex values that may not match Fluent's algorithmic output. Any Fluent UI upgrade could silently change dark theme colors.

**Action**: Per SKILL.md ("the codebase wins"), update `tokens.css` to match TS values for the 4 confirmed color drifts. Add `Cascadia Mono` to the TS font stack or remove it from CSS.

## Recommended ordered fix list (cheapest to highest-impact)

### Group 1 — Quick wins (< 30 min each)
1. **Fix tokens.css drift** — update 4 color values + font-mono to match TS codebase
2. **Remove "Copied!" exclamation** — 1 file, 1 line
3. **Replace emoji/unicode with Fluent icons** — 6 locations, mechanical replacement
4. **Replace 3 inline SVGs with Fluent icons** — AppShell, TabStrip, FileSidebar

### Group 2 — Moderate effort (1-2 hours each)
5. **Add missing ARIA attributes** — 8 elements across 5 files
6. **Replace 8 hardcoded box-shadows with tokens** — 5 dialogs + 3 others
7. **Migrate macOS diag shadow2-on-cards to strokes** — 11 occurrences in 5 files

### Group 3 — Systematic passes (2-4 hours each)
8. **Theme the entire timeline directory** — 25+ hardcoded hex colors across 7 files. This is the single highest-impact fix.
9. **Sentence-case all button labels** — 40+ labels across the codebase. Mechanical but touches many files.
10. **Normalize border-radius to scale** — 50+ occurrences. Decide: add `3px` to the scale, or migrate to `2px`/`4px`.
11. **Normalize spacing to scale** — 50+ occurrences. Decide: add `14px` to the scale, or migrate to `12px`/`16px`.

### Group 4 — Design decisions needed
12. **Dark theme explicit overrides** — Should `theme-dark.ts` pin colors explicitly instead of relying on `createDarkTheme`? (Documented in OPEN-QUESTIONS)
13. **Merge tab color palette** — LogListView uses 8 hardcoded colors for merge tabs. Need a semantic token set or a fixed palette in the theme.
14. **`borderRadius: 3px` proliferation** — 30+ uses. Is this an intentional "small badge" size that should be added to the scale, or should it be `2px`/`4px`?
